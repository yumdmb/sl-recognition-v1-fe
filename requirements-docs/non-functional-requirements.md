4.1.2 Non-Functional Requirements 

Non-functional requirements define the system's operational qualities, ensuring that it performs efficiently, securely, and reliably under different conditions. These requirements focus on aspects such as performance, scalability, security, usability, and compliance. The purpose of non-functional requirements is to ensure that the SignBridge system not only meets functional expectations but also provides a seamless, secure, and user-friendly experience. 

| ID | Type | Non-Functional Requirements |
|----|------|----------------------------|
| NFR-001 | Performance | The system shall recognize a gesture and return results within 5 seconds for real-time input. |
| NFR-002 | Security | The system must be able to give access to authorized user only. |
| NFR-003 | Usability | Users shall be able to switch languages (English/Malay) with a simple toggle. |
| NFR-004 | Portability | The system shall be compatible with various intended devices or platforms. |
| NFR-005 | Availability | The system shall be available at all times, except during the scheduled maintenance window. |
